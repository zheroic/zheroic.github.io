数据处理基础

IS NULL/IS NOT NULL :检查空值 eg：WHERE column IS [NOT] NULL

IN:相当于OR，但比OR直观，执行更快。 eg：WHERE column IN [‘a’,’b’]

NOT：否定其后的条件

DISTINCT：作用与后面全部列

DESC：只作用于紧靠其前的列

EXISTS：要么全部都可以，要么全部都失败。如果EXISTS后面的查询结果不为null，则返回全部Select结果，如果为null，则Select返回null

%：通配符，代表搜索模式中给定位置的0个，1个或多个字符，不会匹配null值

_：通配符，只匹配一个字符

CONCAT：拼接字符串 eg：SELECT CONCAT(column1,column2)/SELECT CONCAT(column1,’—‘,column2)

常用函数：

  • RTRIM(column)、LTRIM(column)、TRIM(column): 去除列值右/左/左右 空格

  • LEFT(column):返回字符串左边的字符

  • LENGTH(column)/DATALENGTH()/LEN():返回字符串长度

  • LOWER(column)/UPPER():将字符串转换为小写/大写

  • SOUNDEX():返回字符串的 SOUNDEX值,SOUNDEX考虑了类似的发音字符和音节，使得能对字符串进行发音比较而不是字母比较 。

    • Customers表中有一个顾客Kids Place，其联系名为Michelle Green。但如果这是错误的输入，此联系名实际上应该是Michael Green，该怎么办呢？

    • SELECT cust_name, cust_contact
FROM Customers
WHERE SOUNDEX(cust_contact) =
SOUNDEX('Michael Green')

日期时间处理函数：

  • YERA/MONTH/DAY/HOUR/DATE(datetime_column):获取date字段的年/月/日/小时/日期

  • NOW():获取系统时间

  • 获取当前日期：current_date()/current_date/curdate()

  • 获取当前时间：curtime()/current_time/current_time()

    • 查询一天：
    select * from 表名 where to_days(时间字段名) = to_days(now());

select * from 表名 where date(时间字段名) = curdate();
    • 昨天
    select * from 表名 where to_days( now( ) ) - to_days( 时间字段名) <= 1;
    • 7天
    select * from 表名 where date_sub(curdate(), interval 7 day) <= date(时间字段名);
    • 一周内数据
    select * from 表名 where 时间字段名 between current_date()-7 and sysdate();
    • 近30天
    select * from 表名 where date_sub(curdate(), interval 30 day) <= date(时间字段名);
select * from tbl_name where to_days(now()) - to_days(date_col) <= 30;
    • 本月
    select * from 表名 where date_format( 时间字段名, '%Y%m' ) = date_format(curdate( ) , '%Y%m' )
    • 查询一个月
    select * from table where date_sub(curdate(), interval 1 month) <= date(column_time);
    • 上一月
    select * from 表名 where period_diff( date_format( now( ) , '%Y%m' ) , date_format( 时间字段名, '%Y%m' ) ) =1

数值处理函数:

  • ABS():绝对值
  • COS()、SIN()、TAN():三角函数值
  • EXP(x):e的x次方
  • PI():圆周率
  • SQRT():平方根

聚集函数：

  • AVG():某列平均值，只能作用于单个列，忽略值为null的行
  • COUNT()：某列行数。COUNT(*)计算表的行数，不会忽视带有null值的行除非一行全为null。COUNT(colummn)计算某列具有值的行数，忽略null。
  • MAX()、MIN():忽略null
  • SUM():某列和，忽略null
  • 聚集函数默认不去重，去重则要使用DISTINCT+列名，DISTINCT不能作用于COUNT(*)

分组数据：GROUP BY/HAVING

  • 过滤行：GROUP BY。过滤分组:HAVING,经常结合GROUP BY使用

  • GROUP BY子句中列出的每一列都必须是检索列或有效的表达式（但不能是聚集函数）。
    如果在SELECT中使用表达式，则必须在GROUP BY子句中指定相同的表达式。不能使用别名

  • 除聚集计算语句外，SELECT语句中的每一列都必须在GROUP BY子句中给出

  • 如果分组列中包含具有NULL值的行，则NULL将作为一个分组返回。如果列中有多行NULL值，它们将分为一组。

  • GROUP BY子句必须出现在WHERE子句之后，ORDER BY子句之前。

  • HAVING可以使用聚集函数，GROUP BY不可以

  • 分组和排序：ORDER BY/GROUP BY

    • ORDER BY：对产生的输出排序，任意列都可以使用（甚至非选择的列也可以使用）
    • GROUP BY：对行分组，但输出可能不是分组的顺序。只可能使用选择列或表达式列， 而且必须使用每个选择列表达式。如果与聚集函数一起使用列（或 表达式），则必须使用

SELECT字句编写顺序：

SELECT…FROM…WHERE…GROUP BY…HAVING…ORDER BY…

子查询：作为子查询的SELECT只能查询单列，检索多列则会错误。（例子SQL必知必会第十一课)

联结表：（例子SQL必知必会第十二课)

  • 叉联结：没有联结条件，返回笛卡尔积
  • 内联结（等值联结）：基于两个表之间的相等测试（INNER JOIN…ON 条件)

创建高级联结：（使用表别名，简化）(例子SQL必知必会第十三课)

  • 自联结（self-join）：SELECT语句使用表名前缀明确给出所需列的全名。如果不这样，DBMS将返回
    错误，因为列各有两个。DBMS不知道想要的是哪一列（即使它们其实是同一列）。
  • 自然联结（nature-join）：标准的联结（前一课中介绍的内联结）返回所有数据，相同的列甚至多次出现。自然联结排除多次出现，使每一列只返回一次。 手动去重。
  • 外联结（outer-join）：联结包含了那些在相关表中没有关联行的行。这种联结称为外联结。
    • right/left-join: RIGHT/LEFT JOIN…ON 条件
    • full-join:MySQL不支持

组合查询：UNION/UNION ALL

  • UNION:自动去重
  • 在用UNION组合查询时，只能使用一条ORDER BY子句，它必须位于最后一条SELECT语句之后。

插入：

插一条：
INSERT INTO tablename[(column1,column2,...)] VALUES(value1,value2,...);
插多行:
INSERT INTO tablename[(column1,column2,...)]
SELECT ...;

复制表：

CREATE TABLE tablename AS 
SELECT ...

更新删除:

一次更新一条，多条更新foreach或者case when实现
UPDATE tablename SET column=... [WHERE ...] 
DELETE FROM tablename [WHERE...]
UPDATE可删除指定列（设置值为null），DELETE删除行，不删除表

创建、操纵表：

  • 使用null/not null
  • 使用默认值 default
CREATE TABLE tablename(
column1 datatype [NULL/NOT NULL] [CONSTRAINTS],
column2 datatype [NULL/NOT NULL] [CONSTRAINTS],
...);

创建视图VIEW：

  • 不提供任何的列或者数据，包含的只是一个查询
  • 优点：
    • 重用SQL语句
    • 简化复杂的SQL操作
    • 使用表的一部分而不是整个表
    • 保护数据
  • 限制、规则：
    • 视图命名必须唯一
    • 不能索引，不能有关联的触发器和默认值
CREATE VIEW viewname AS
SELECT column1,column2,..
FROM tablename
[WHERE...] [GROUP BY ...] [HAVING...];

DROP VIEW viewname;

更新表：ALTER

ALTER TABLE tablename 
ADD COLUMN columnname datatype [NULL/NOT NULL] [CONSTRAINTS],
ADD COLUMN columnname datatype [NULL/NOT NULL] [CONSTRAINTS],
...
DROP COLUMN columnname,
DROP COLUMN columnname,
...
;

删除表：

DROP TABLE tablename

重命名表：

RENAME TABLE oldname TO newname

存储过程：

  • 优点：简单、安全、高性能

    • 简化对变动的管理
    • 因为存储过程通常以编译过的形式存储，所以DBMS处理命令的工作较少，提高了性能
    • 存在一些只能用在单个请求中的SQL元素和特性，存储过程可以使用它们来编写功能更强更灵活的代码

  • 缺陷：

    • 编写真正的可移植存储过程几乎是不可能的。
    • 编写存储过程比编写基本SQL语句复杂

  • 执行：

    EXCUTE procedurename(arg1,arg2,...)

事务处理：transaction

事务（transaction）指一组SQL语句；
回退（rollback）指撤销指定SQL语句的过程；
提交（commit）指将未存储的SQL语句结果写入数据库表；
保留点（savepoint）指事务处理中设置的临时占位符（placeholder），可以对它发布回退（与回退整个事务处理不同）。

  • rollback/commit savepoint:保留点越多越好，可以灵活回退

  • 可回退：update/delete/insert 不可回退：drop/create

  • 开始事务：
START TRANSACTION
...
使用rollback：
DELETE FROM Orders;
ROLLBACK;
使用commit：
COMMIT TRANSACTION；
使用savepoint：
SAVE TRANSACTION delete1;
...
ROLLBACK TO delete1;

游标：CURSOR

  • 能够标记游标为只读，使数据能读取，但不能更新和删除。
  • 能控制可以执行的定向操作（向前、向后、第一、最后、绝对位置、相对位置等）。
  • 能标记某些列为可编辑的，某些列为不可编辑的。
  • 规定范围，使游标对创建它的特定请求（如存储过程）或对所有请求可访问。
  • 指示DBMS对检索出的数据（而不是指出表中活动数据）进行复制，使数据在游标打开和访问期间不变化。 
DECLARE cursorname CURSOR FOR SELECT...;
OPEN cursorname;
FETCH cursorname INTO columnname;
CLOSE cursorname;

高级SQL特性：

  • 约束：

    • 主键：值唯一，没有null值，主键列从不修改/更新

      • 自增主键：eg:
ALTER TABLE tablename ADD PRIMARY KEY(id);
ALTER TABLE tablename CHANGE id id int(10) not null auto_increament=1
删除主键：先删自增如果有，再删主键
ALTER TABLE tablename CHANGE id id int(10);
ALTER TABLE tablename DROP PRIMARY KEY;

    • 外键：

      • ALTER TABLE tablename ADD CONSTRAINT FK_ID FOREIGN KEY(外键字段名) REFERENCE 外表表名(对应字段名);

    • 唯一约束：允许null

    • 检查约束：CHECK(条件)。检查最大最小值，指定范围，只允许特定值。

  • 索引：

    • CREATE INDEX indexname ON tablename(columnlist);

  • 触发器：约束比触发器更快

    • 保持数据的一致（INSERT，UPDATE)
    • 计算列的值或者更新时间戳
