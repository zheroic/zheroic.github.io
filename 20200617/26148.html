递归优化（斐波那契数列为例）

递归　

关于递归的概念，我们都不陌生。简单的来说递归就是一个函数直接或间接地调用自身，是为直接或间接递归。一般来说，递归需要有边界条件、递归前进段和递归返回段。当边界条件不满足时，递归前进；当边界条件满足时，递归返回。用递归需要注意以下两点：

(1) 递归就是在过程或函数里调用自身。

(2) 在使用递归策略时，必须有一个明确的递归结束条件，称为递归出口。

递归一般用于解决三类问题：

　 (1)数据的定义是按递归定义的。（Fibonacci函数，n的阶乘）

　 (2)问题解法按递归实现。（回溯）

　 (3)数据的结构形式是按递归定义的。（二叉树的遍历，图的搜索）

递归的缺点：

　　递归解题相对常用的算法如普通循环等，运行效率较低。因此，应该尽量避免使用递归，除非没有更好的算法或者某种特定情况，递归更为适合的时候。在递归调用的过程当中系统为每一层的返回点、局部量等开辟了栈来存储，因此递归次数过多容易造成栈溢出。

线性递归/普通递归

斐波那契数列的标准公式为：F(1)=1，F(2)=1, F(n)=F(n-1)+F(n-2)（n>=3，n∈N*）

以计算斐波那契数列为例，求斐波那契数列的第n项。程序代码如下：

public class Fibonacci {
    /**
     * 数列从0开始，0，1，1，2，3...
     * @param n
     * @return
     */
    public static long fib(int n){
        if(n<0){
            return -1;
        }
        if(n==0){
            return 0;
        }
        if(n==1){
            return 1;
        }
        return fib(n-1)+fib(n-2);
    }

代码看起来很优雅简洁，但其运行效率奇低，运行测试代码

public static void main(String[] args) {

        long begin = System.currentTimeMillis();
        long res=  fib(50);
        long end = System.currentTimeMillis();
        System.out.println("fib(50)="+res+"，耗时"+(end-begin)+"毫秒");
    }

fib(50)==12586269025，耗时62608毫秒

优化递归

用数组保存结果，避免重复计算

空间换时间，把计算出来的fib(n)存储起来，不用重复计算，空间复杂度O(n)

 /**
     * 数列从0开始，0，1，1，2，3...
     * @param n 
     * @return
     */
public static long fibArray(int n){
        if(n==0){
            return 0;
        }
        if(n==1){
            return 1;
        }
        long[] array =new long[n+1];
        array[0]=0;
        array[1]=1;
        for(int i=2;i<n+1;i++){
            array[i]=array[i-1]+array[i-2];
        }
        return array[n];
    }

fibArray(50)=12586269025，耗时0毫秒

优化存储

只存储3个数：前两个数和前两个数相加计算出来的结果，一直向前滚动

public static long  fibBetterSave(int n){
        long first =0;
        long second =1;
        long sum = 0;

        for(int i=2;i<=n;i++){
            sum=first+second;
            first=second;
            second=sum;
        }
        return sum;
    }

fibBetterSave(50)=12586269025，耗时0毫秒

尾递归

递归是从大到小n,n-1,n-2…1，尾递归是从小到大1,2…n-1,n

采用尾递归算法，可以极大地提高运行效率

如果一个函数中所有递归形式的调用都出现在函数的末尾，我们称这个递归函数是尾递归。需要说明的是，只有当递归调用是整个函数体中最后执行的语句且它的返回值不属于表达式的一部分时，这样的递归调用才是尾递归。

 /**数列从0开始，0，1，1，2，3...
     * @param n
     * @param res1
     * @param res2
     * @return
     */
    public static long fibonacciTailRecursive(int n,long res1,long res2){

        if(n==0){
            return res1;
        }
        return fibonacciTailRecursive(n-1,res2,res1+res2);
    }

运行测试代码：

public static void main(String[] args) {
        long begin = System.currentTimeMillis();
        long res=  fibonacciTailRecursive(50,0,1);
        long end = System.currentTimeMillis();
        System.out.println("fibonacciTailRecursive(50)="+res+"，耗时"+(end-begin)+"毫秒");
    }

fibonacciTailRecursive(50)=12586269025，耗时0毫秒
